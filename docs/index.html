Quick start

You just have to include the pocketgrid.css (or pocketgrid.min.css) in the <head> tag of your page
and don't forget the viewport meta tag!

Here is the minimal markup you need to put in your pages:

<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="pocketgrid.css" />
  </head>
  <body>
    <!-- YOUR CONTENT HERE -->
  </body>
</html>

Then, you just have to add block classes to elements you want to put in the grid, and block-group classes for block containers.

Basic rules

The basic rules with PocketGrid are:

  • Blocks have a default width of 100%
  • Blocks must be contained in a block-group
  • Blocks are stacked by default (ideal for mobile-first design)
  • To display blocks side-by-side (like columns), just give them a width in percentage (for example 30% and 70% to make 2 columns)
  • Blocks automatically go to a new row if the current row is full! (so you don't need to put a block-group container for each row)

Offsets

You can add offsets to blocks (to simulate empty columns for example).
To do so, you just have to set the margin-left or margin-right properties of a block to the size of one or more columns.
For example, here the "Left offset" block has a left offset of 1 column (margin-left: 20%) and the "Right offset" block has a left offset of 2 column (margin-right: 40%)
If you look at the HTML source, the "BEFORE" column is declared before the "AFTER" column, whereas it is displayed after.

Push / Pull (swap columns)

To change the order of columns (name them A and B), you have to :
  • Set the position property of the blocks to "relative"
  • Set the left property of A to the width of B
  • Set the left property of B to minus the width of A
If you look at the HTML source, the "BEFORE" column is declared before the "AFTER" column, whereas it is displayed after.

Fix for different block heights

When blocks do not have the same height, your layout can seem broken. For example, in the demo below, we want to display 5 blocks per row, but the 3rd block has a greater height than the other blocks.
With the PocketGrid "automatic row" feature, "Block 6" should be placed on a brand new row, but instead, it is blocked by the big "Block 3". This is because PocketGrid uses CSS floats to manage blocks.
To solve this problem, just put "clear: left" in the style of the block ("Block 6" here) to force using a real new row, no matter the height of the previous blocks.
